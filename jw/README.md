# 12_pjt. SPA 구조화 및 상태 관리

본 프로젝트에서는 아래의 4가지 핵심 필수 요구사항을 성공적으로 구현 및 통합했습니다.
1. **SPA 구조화 및 상태 관리**: `frontend`와 `backend` 폴더를 분리하고, Vue Router 기반의 SPA 환경과 Pinia를 활용한 전역 상태 관리 도입
2. **커뮤니티(게시판) 기능**: Django(백엔드)와 Vue(프론트엔드)를 연동하여 맞춤형 여행 코스 자랑 및 댓글 소통이 가능한 커뮤니티 게시판 구축
3. **모달 UI 및 외부 API 시각화**: `<Teleport>`를 활용한 독립적인 상세 정보 모달 팝업 구현 

---

## 1. 학습한 내용 및 어려웠던 부분

- **Vue Router 및 상태 관리 (Pinia) 구조화**: 단일 애플리케이션(SPA) 구조를 갖추기 위해 프론트엔드와 백엔드를 물리적 폴더 단위로 분리하고, Vue Router를 통해 컴포넌트 간 전환을 매끄럽게 구성하는 방법을 학습했습니다. 더불어 Pinia를 도입하여 전역 상태(사용자 인증 정보, 여행 코스 데이터 등)를 체계적으로 관리하는 패턴을 익혔습니다.
- **모달 UI 및 외부 API 시각화 확장**: 특정 여행지 상세 정보를 렌더링할 때 Vue의 `<Teleport>` 기능을 활용하여 부모 컴포넌트의 CSS 종속성(z-index, backdrop-filter 등)을 벗어난 쾌적한 오버레이 모달 팝업을 띄우는 기법을 학습했습니다.

---

## 2. 새로 배운 것들 및 느낀 점

- **Vue의 `<Teleport>`가 주는 자유로움**
  그동안 모달이나 팝업창을 구현할 때마다 z-index 싸움이나 부모 태그의 CSS(`transform` 등) 제약 때문에 레이아웃이 꼬이는 스트레스가 컸습니다. 이번에 `<Teleport to="body">`를 적용하면서 Vue가 렌더링 위치를 DOM 트리 최상단으로 옮겨주는 기능의 편리함과 강력함을 새로 배웠습니다.
